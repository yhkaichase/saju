# calendar — 음양력 변환 · 절기(節氣) 계산

> ⚠️ 이 모듈의 계산은 **정확성이 최우선**이며 추정 구현을 금지합니다.
> 구현 시 반드시 출처·정밀도·시간대를 코드 주석과 PR 본문에 남기세요.
> (규칙: `../../../.claude/rules/saju-domain.md`)

## 구현 완료

- **`julian-day.ts` — `gregorianToJulianDayNumber(y, m, d)`**: 정수 JDN(Fliegel–Van
  Flandern, proleptic Gregorian). 일주(日柱) 계산의 기반. KST 벽시계 Y/M/D를
  입력해야 하며 UTC 게터 사용 금지(자정 근처 하루 밀림). 일주 자체는
  `src/lib/saju/day-pillar.ts`에 구현(앵커: 1949-10-01=甲子, 공식 `(JDN+49)%60`).
  일자 경계 정책(자정/자시)은 학파 분기 사안으로 옵션화되어 있음.

## 구현 예정 (근거 필요)

- **절기 시각 계산**: 태양황경(黃經)이 15°의 배수가 되는 순간.
  - 후보: 순수 TS 천문 라이브러리(예: `astronomy-engine`, Meeus 알고리즘 기반).
    pyswisseph 수준의 각초 정밀도는 불필요 — 사주는 **분(分) 단위** 정밀도면 충분.
  - 입춘(立春) = 황경 315°. 연주(年柱) 경계.
  - 12절기 경계로 월주(月柱)를 결정.
- **음양력 변환**: 양력 ↔ 음력. 윤달 처리 포함.
  - 한국 사주의 ground truth는 **KASI(한국천문연구원) 공식 만세력/절기 시각**입니다.
    외부 천문엔진 결과는 KASI 공식값과 분 단위로 어긋날 수 있으므로,
    **반드시 KASI 데이터로 골든 테스트를 고정**한 뒤 사용합니다.
- **시간대 정책**: KST(UTC+9) 기준, 진태양시(眞太陽時) 보정 여부를 명시.

## 테스트 원칙

- 입춘 당일, 자시 경계(23:30/00:30), 윤달 케이스를 골든 테스트에 반드시 포함.
- 테스트 데이터의 출처(KASI 등)를 주석으로 기록.
